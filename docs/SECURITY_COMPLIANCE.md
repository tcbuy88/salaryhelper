# 安全与合规要点（概览）

1. 认证与传输
- 强制 HTTPS/TLS（TLS1.2+）
- 身份验证用 JWT（短期）+ refresh token（长期）
- Admin 路径启用 MFA（企业微信/OTP）

2. 敏感字段保护
- 字段级加密：身份证号、银行卡号等（使用 KMS）
- 文件存储：私有 bucket + 短时签名 URL
- 元数据审计：记录上传者、时间戳、hash

3. 数据保留与最小化
- 只保留业务必需字段
- 提供用户删除/导出接口
- 保留策略与法务确认（如证据 7 年）

4. 模型与内容治理
- 模型输出加免责声明（UI 显示）
- 保存 prompt、检索来源、token 统计供审计
- 高风险回答触发人工复核

5. 录音合规
- 录音前强制确认並记录用户同意
- 录音加密存储、访问受限

6. 审计与权限
- 关键操作写结构化日志（ELK）
- RBAC：user/reviewer/lawyer/admin
